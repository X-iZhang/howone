# HowOne AI Prompt Guide

Writing effective prompts is crucial for generating high-quality apps on HowOne AI.

## Prompt Structure

Use this template for best results:

```
Create an application where users can [USER ACTIONS], 
and the AI can [AI FUNCTIONS].
The app should [ADDITIONAL FEATURES].
```

## Prompt Examples by App Type

### Data Analysis Apps

```
Create an application where users can upload CSV files with sales data,
and the AI can analyze trends, identify top products, and generate
monthly reports with charts. Include filtering by date range and
export to PDF functionality.
```

### Content Generation Apps

```
Create an application where users can enter a topic and target audience,
and the AI can generate blog posts, social media captions, and email
newsletters. Include tone adjustment (formal/casual) and length controls.
```

### Management/Tracking Apps

```
Create a pet hospital management application that not only records
pet care details but also generates health reports with one click
based on those records. Include appointment scheduling and
medication reminders.
```

### Education Apps

```
Create an application for users to enter disciplines, wrong questions,
and wrong answers. AI can automatically disassemble the loopholes of
knowledge points, generate analysis of wrong questions, and generate
AI reports based on the record of wrong questions in the user's history.
```

### Goal/Project Management Apps

```
Generate an application for goal disassembly and monitoring. Users can
enter the goal and expected achievement time. AI can help users generate
a list of plans with implementation steps, and each implementation step
has the final completion time, the content to be completed, and the
implementation guidance.
```

### Contract/Legal Apps

```
Create an application where users can copy the contract text, let AI
analyze and highlight potential risk terms, unclear statements and notes,
etc., and explain them in easy-to-understand language.
```

## Key Elements to Include

1. **User Input**: What will users provide? (text, files, images, etc.)
2. **AI Output**: What should the AI produce? (analysis, content, reports)
3. **Features**: Any specific functionality? (filtering, export, scheduling)
4. **Format**: How should results be presented? (charts, tables, documents)

## Common Mistakes to Avoid

| ❌ Bad | ✅ Good |
|--------|---------|
| "Make an app for me" | "Create a task management app where users can..." |
| "Something with AI" | "AI should analyze the input and generate a report" |
| "Like that other app" | "Features: user profiles, dashboard, notifications" |
| Too vague | Specific inputs, outputs, and features |

## Tips for Better Results

1. **Be specific about data flow**: "Users upload X → AI processes Y → Output Z"
2. **Mention UI preferences**: "Include a dashboard", "Use cards layout"
3. **Specify integrations**: "Connect to Google Calendar", "Export to PDF"
4. **Describe the user**: "For small business owners", "For students"

## Iteration Strategy

If the first generation isn't perfect:

1. Use the chat panel to refine: "Add a filter for date range"
2. Select specific elements and edit them
3. Use "Edit Workflow" to modify backend logic
4. Trigger self-evolution with real data for optimization
